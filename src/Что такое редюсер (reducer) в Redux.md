Редюсер (reducer) в Redux - это чистая функция, которая принимает текущее состояние приложения и действие (action) и возвращает новое состояние. Он определяет, как изменить состояние приложения в ответ на действие.

Функция-редюсер имеет следующую сигнатуру:

```javascript
const reducer = (state = initialState, action) => {
  // Логика обработки действия и возврата нового состояния
};
```

Главные принципы работы редюсера в Redux:

1. Чистота: Редюсер должен быть чистой функцией, что означает, что он должен возвращать новый объект состояния, не изменяя исходное состояние и без побочных эффектов. Это позволяет Redux эффективно отслеживать изменения и контролировать поток данных.

2. Неизменность состояния: Редюсер должен создавать новый объект состояния при каждом изменении, а не изменять текущий объект состояния напрямую. Это гарантирует, что предыдущие состояния сохраняются и можно отслеживать историю изменений.

3. Комбинирование редюсеров: В больших приложениях часто требуется использовать несколько редюсеров для управления разными частями состояния. Эти редюсеры могут быть объединены вместе с помощью функции `combineReducers` из пакета `redux`. Это облегчает масштабирование и организацию кода.

Пример простого редюсера, который обрабатывает действие инкремента счетчика:

```javascript
const counterReducer = (state = 0, action) => {
  switch (action.type) {
    case 'INCREMENT_COUNTER':
      return state + action.payload;
    default:
      return state;
  }
};
```

В этом примере редюсер `counterReducer` принимает текущее состояние `state` (начальное значение установлено в 0) и действие `action`. Внутри редюсера используется оператор `switch`, чтобы определить, как обработать определенное действие. Если тип действия соответствует `'INCREMENT_COUNTER'`, то редюсер возвращает новое состояние, увеличенное на значение `action.payload`. В противном случае возвращается текущее состояние без изменений.

Важно помнить, что каждый редюсер должен возвращать новый объект состояния, даже если состояние не изменяется. В противном случае Redux не сможет корректно отслеживать изменения и обновлять компоненты, подписанные на состояние.

Редюсеры являются ключевыми элементами в паттерне Flux и Redux, и они обеспечивают предсказуемость и контролируемость состояния в приложении.
